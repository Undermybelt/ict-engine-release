下面这 12 篇是我会拿去支撑你这套 **canonical belief chain** 的最小硬核文献集。重点不是“预测收益”，而是把 `node / branch / scenario / path` 的 prior、posterior、source weighting、ranking calibration 从 heuristic 推到 principled update。

## 1. Dynamic Bayesian Networks: Representation, Inference and Learning

**Citation**
Kevin P. Murphy. PhD thesis, UC Berkeley EECS, 2002. Link: UC Berkeley report page. ([EECS at UC Berkeley][1])

**Why it matters**
给你的 `node / branch / scenario / path` 一个正式 DBN 表达：每个 time slice 是结构状态，跨 slice 是 transition，观测证据进入 likelihood。

**Exact mechanism to steal**
把 belief state 写成：

[
P(Z_{1:T}, E_{1:T})=P(Z_1)\prod_{t=1}^T P(E_t|Z_t)\prod_{t=2}^T P(Z_t|Z_{t-1})
]

其中：

[
Z_t=(node_t, branch_t, scenario_t, path_t)
]

在线更新：

[
Bel_t(z) \propto P(E_t|z)\sum_{z'}P(z|z')Bel_{t-1}(z')
]

这里 `E_t` 可以是 live feedback、artifact validation、backtest、research、mutation、analyze 的统一 evidence packet。

**Suggested repo target**
`structural_prior_state`

**Risk / mismatch**
Murphy 是通用 DBN，不会帮你定义 trading-specific node taxonomy；状态空间如果不剪枝，会爆炸。

## 2. A New Approach to the Economic Analysis of Nonstationary Time Series and the Business Cycle

**Citation**
James D. Hamilton. *Econometrica*, 1989. DOI: 10.2307/1912559. ([Semantic Scholar][2])

**Why it matters**
这是 regime-switching posterior filtering 的硬源头：把不可见 regime 当作离散 Markov state，用观测序列实时更新 regime posterior。

**Exact mechanism to steal**
Hamilton filter：

[
\xi_{t|t}(j)=
\frac{
f(y_t|s_t=j,Y_{t-1})
\sum_i p_{ij}\xi_{t-1|t-1}(i)
}{
\sum_j f(y_t|s_t=j,Y_{t-1})
\sum_i p_{ij}\xi_{t-1|t-1}(i)
}
]

工程映射：

[
s_t=node_t,\quad p_{ij}=P(node_j|node_i)
]

`branch` 就是 transition row 上的 candidate edge。

**Suggested repo target**
`BBN node/branch posterior update`

**Risk / mismatch**
Hamilton 原模型偏宏观时间序列，默认低维、固定 regime 数；你的 path/scenario 层更细，需要 hierarchical extension。

## 3. A sticky HDP-HMM with application to speaker diarization

**Citation**
Emily B. Fox, Erik B. Sudderth, Michael I. Jordan, Alan S. Willsky. *Annals of Applied Statistics*, 2011. DOI: 10.1214/10-AOAS395. ([Project Euclid][3])

**Why it matters**
解决 structural node “过度跳变”的问题。你的系统已有 break penalty，这篇给 principled sticky transition prior。

**Exact mechanism to steal**
sticky transition prior：

[
\pi_j|\beta \sim DP\left(\alpha+\kappa,\frac{\alpha\beta+\kappa\delta_j}{\alpha+\kappa}\right)
]

其中 (\kappa) 是 self-transition bias。工程上：

[
P(node_t=node_{t-1}) \uparrow \quad \text{when break evidence weak}
]

把 `break penalty` 从 heuristic 改成 transition prior 中的 (\kappa)。

**Suggested repo target**
`BBN node/branch posterior update`

**Risk / mismatch**
HDP-HMM 是非参数 HMM，推断成本较高；工程里可能只需要 sticky finite-state transition prior，不需要完整 HDP。

## 4. Sequential Bayesian Learning for Hidden Semi-Markov Models

**Citation**
Patrick Aschermayr, Konstantinos Kalogeropoulos. arXiv, 2023. DOI/arXiv: 10.48550/arXiv.2301.10494. ([arXiv][4])

**Why it matters**
HSMM 比 HMM 更适合你的 structural node，因为 node/scenario 往往有 dwell time，不应每根 bar 都等概率转移。

**Exact mechanism to steal**
把 latent state 扩成：

[
Z_t=(state_t, duration_t)
]

transition 不再只是 (P(s_t|s_{t-1}))，而是：

[
P(s_t,d_t|s_{t-1},d_{t-1})
]

如果 duration 未耗尽，状态延续；如果 duration 到期，才抽新 state。工程上可给每个 node / scenario 一个 duration prior：

[
d \sim g_{node}(d)
]

例如 squeeze、trend continuation、mean reversion 各自有不同 half-life。

**Suggested repo target**
`structural_prior_state`

**Risk / mismatch**
需要你定义 duration distribution；如果 node taxonomy 不稳定，duration prior 会漂。

## 5. Online Learning of Order Flow and Market Impact with Bayesian Change-Point Detection Methods

**Citation**
Ioanna-Yvonni Tsaknaki, Fabrizio Lillo, Piero Mazzarisi. *Quantitative Finance*, 2025, 25(2):307–322. DOI: 10.1080/14697688.2024.2337300. ([IDEAS/RePEc][5])

**Why it matters**
这是最贴近 execution system 的一篇：显式把 order-flow regime、change point、market impact、online prediction 连起来，不是 generic next-return forecasting。论文说明用 BOCPD 实时识别 order-flow regime shift，并用于 order flow / market impact prediction。([arXiv][6])

**Exact mechanism to steal**
BOCPD run-length posterior：

[
P(r_t=0,x_{1:t})=\sum_{r_{t-1}}P(r_{t-1},x_{1:t-1})H(r_{t-1})P(x_t|r_{t-1})
]

[
P(r_t=r_{t-1}+1,x_{1:t})
=P(r_{t-1},x_{1:t-1})(1-H(r_{t-1}))P(x_t|r_{t-1})
]

工程映射：

* `r_t` = 当前 structural node 已持续多久
* hazard (H(r)) = node break prior
* high change-point posterior = branch expansion / invalidation trigger

**Suggested repo target**
`BBN node/branch posterior update`

**Risk / mismatch**
论文聚焦 order flow / market impact，不直接覆盖你的 scenario / playbook 层；但 BOCPD 机制非常适合 node break / branch birth。

## 6. Power Prior Distributions for Regression Models

**Citation**
Joseph G. Ibrahim, Ming-Hui Chen. *Statistical Science*, 2000, 15(1):46–60. DOI: 10.1214/ss/1009212673. ([Project Euclid][7])

**Why it matters**
这是把 offline evidence 变成 prior 的最直接机制。你的 `analyze / research / backtest / mutation / artifact validation` 都可以作为 historical data (D_0)，但用 power (a_0) 控制借用强度。

**Exact mechanism to steal**

[
\pi(\theta|D_0,a_0)\propto L(\theta|D_0)^{a_0}\pi_0(\theta)
]

其中：

[
a_0 \in [0,1]
]

工程映射：

[
a_0 = source_weight \times quality_calibration \times recency_decay
]

对 Beta-Binomial path success prior：

[
\alpha_p=\alpha_0+\sum_s a_s \cdot success_{s,p}
]

[
\beta_p=\beta_0+\sum_s a_s \cdot failure_{s,p}
]

**Suggested repo target**
`artifact-validation prior source`

**Risk / mismatch**
Power prior 假设 historical evidence 与当前 task 同分布或近似可借用；market regime shift 下必须配 recency decay / drift gate。

## 7. Maximum Likelihood Estimation of Observer Error-Rates Using the EM Algorithm

**Citation**
A. P. Dawid, A. M. Skene. *Journal of the Royal Statistical Society: Series C / Applied Statistics*, 1979, 28(1):20–28. DOI: 10.2307/2346806. ([Wiley Online Library][8])

**Why it matters**
你的 evidence sources 不是同质观测器。Dawid-Skene 给“source reliability / source confusion matrix”机制：不是简单手写权重，而是学习每个 source 在不同 latent truth 下的偏差。

**Exact mechanism to steal**
latent true class：

[
z_i \sim Categorical(\pi)
]

source (j) 的 observation：

[
y_{ij}\sim Categorical(\theta_{j,z_i})
]

marginal likelihood：

[
P(y_i|\theta,\pi)=
\sum_k \pi_k\prod_j \theta_{j,k,y_{ij}}
]

工程映射：

* `live feedback`、`artifact validation`、`backtest`、`research`、`mutation`、`analyze` 都是 annotator/source
* (\theta_{j,k}) 是 source 在 true path outcome = (k) 时的 confusion profile
* source weight 从固定优先级升级为 learned reliability + prior hierarchy

**Suggested repo target**
`live feedback posterior update`

**Risk / mismatch**
原文是多人标注/诊断模型，不是时间序列；需要加时间衰减，否则老 source reliability 会滞后。

## 8. Using Bayesian Model Averaging to Calibrate Forecast Ensembles

**Citation**
Adrian E. Raftery, Tilmann Gneiting, Fadoua Balabdaoui, Michael Polakowski. *Monthly Weather Review*, 2005. DOI: 10.1175/MWR2906.1. ([Semantic Scholar][9])

**Why it matters**
你的 offline sources 本质上是一组 probabilistic forecasters。BMA 给出“按历史 predictive skill 加权组合”的 calibrated mixture，不是硬编码 source order。

**Exact mechanism to steal**

[
p(y|x,D)=\sum_m w_m p_m(y|x,D_m)
]

[
w_m \propto p(m)\prod_{\tau \in W}p_m(y_\tau|x_\tau)
]

工程映射：

* (m) = analyze / research / backtest / mutation / artifact validation / live feedback
* (w_m) = source posterior reliability
* (W) = rolling calibration window
* final prior = weighted ensemble prior

**Suggested repo target**
`structural_prior_state`

**Risk / mismatch**
BMA 假设每个 source 能输出可比较的 predictive density；你的 mutation/analyze 可能只输出 score，需要先标准化成 likelihood 或 pseudo-likelihood。

## 9. Adaptive Conformal Inference Under Distribution Shift

**Citation**
Isaac Gibbs, Emmanuel Candès. NeurIPS / arXiv, 2021. DOI/arXiv: 10.48550/arXiv.2106.00170. ([arXiv][10])

**Why it matters**
你已有 conformal coverage 作为 quality calibration。ACI 给 online drift 下的 coverage maintenance，不要求 exchangeability 固定成立；论文明确针对 online setting 和 changing data-generating distribution。([arXiv][10])

**Exact mechanism to steal**
维护一个动态 miscoverage level：

[
\alpha_{t+1}=\alpha_t+\gamma(\alpha^* - err_t)
]

其中：

[
err_t = \mathbf{1}{y_t \notin C_t}
]

工程映射：

* 如果 recent path outcomes 经常落在 prediction set 外，降低 effective confidence
* ranking score 不只看 CatBoost score，还乘 calibration reliability：

[
rank_score = model_score \times calibration_multiplier
]

**Suggested repo target**
`CatBoost path ranking target`

**Risk / mismatch**
Conformal 处理 coverage，不直接处理 utility / PnL；需要把 outcome label 设计成 path success、stop hit、invalidated、partial compliance 等结构化事件。

## 10. Modeling Delayed Feedback in Display Advertising

**Citation**
Olivier Chapelle. KDD, 2014, pp. 1097–1105. DOI: 10.1145/2623330.2623634. ([ACM Digital Library][11])

**Why it matters**
你的 path outcome 是 delayed reward：entry、confirmation、stop、invalidation、target hit 都可能滞后。Chapelle 的核心是把“尚未发生”区分成 true negative 和 censored positive。

**Exact mechanism to steal**
对 path (i)：

* conversion / success probability: (p_i)
* delay CDF: (F_i(t))
* elapsed time: (e_i)

如果已经成功，likelihood：

[
p_i f_i(d)
]

如果尚未成功：

[
(1-p_i)+p_i(1-F_i(e_i))
]

工程映射：

* 未触发 target 不等于失败
* stop / invalidation / no-confirmation 都应作为 censored or competing-risk outcome
* CatBoost training target 要加 `maturity_mask`

**Suggested repo target**
`CatBoost path ranking target`

**Risk / mismatch**
广告 conversion 是单事件；交易 path 是多阶段事件，需要扩展为 trigger → confirmation → execution → exit 的 survival chain。

## 11. Counterfactual Risk Minimization: Learning from Logged Bandit Feedback

**Citation**
Adith Swaminathan, Thorsten Joachims. ICML / PMLR, 2015. ([Proceedings of Machine Learning Research][12])

**Why it matters**
你说 CatBoost 只在 candidate paths 里 ranking；但历史反馈只来自被展示/被执行/被用户遵守的 paths。CRM 给 logged bandit feedback 的 propensity correction。

**Exact mechanism to steal**
IPS risk：

[
\hat R_{IPS}(\pi)=
\frac{1}{n}\sum_i
\frac{\pi(a_i|x_i)}{p_i}
\Delta(a_i,y_i)
]

CRM 加 variance penalty：

[
\hat R_{CRM}=\hat R_{IPS}+\lambda\sqrt{\frac{\widehat{Var}_{IPS}}{n}}
]

工程映射：

* log `candidate_path_set`
* log `shown_path`
* log `executed_path`
* log `execution propensity`
* 用 propensity-weighted loss 训练 ranking target

**Suggested repo target**
`CatBoost path ranking target`

**Risk / mismatch**
需要准确 logging propensities；如果历史系统没记录 candidate exposure / user compliance probability，早期只能用 approximate propensity。

## 12. Unbiased Learning-to-Rank with Biased Feedback

**Citation**
Thorsten Joachims, Adith Swaminathan, Tobias Schnabel. WSDM, 2017. DOI: 10.1145/3018661.3018699. ([ir.webis.de][13])

**Why it matters**
你的 live feedback 有 selection bias：用户只会看到、选择、执行部分 paths。LTR 的 counterfactual framework 直接对应 path ranking under partial compliance。

**Exact mechanism to steal**
把 pairwise/listwise ranking loss 做 propensity weighting：

[
L_{unbiased}
============

\sum_i
\frac{o_i}{q_i}
\ell(rank_i, y_i)
]

其中：

* (o_i) = observed feedback indicator
* (q_i) = path 被观察 / 被执行 / 被反馈的 propensity
* (y_i) = realized path outcome

工程映射：

[
q_i=P(observed_i|shown_i, user, market_state)
]

训练时不要把 unobserved path 当 negative，而是做 inverse-propensity correction。

**Suggested repo target**
`CatBoost path ranking target`

**Risk / mismatch**
搜索排序的 click bias 不等于交易执行偏差；你需要定义 execution exposure、user compliance、market fillability 三种 propensity。

### 1. `structural_prior_state`

先看：

* **pgmpy/pgmpy**
* **mattjj/pyhsmm**

```text
Z_t = {
  node_t,
  branch_t,
  scenario_t,
  candidate_paths_t,
  node_duration_t,
  transition_prior_t
}
```

pgmpy 负责结构图；statsmodels/Hamilton filter 负责 node posterior baseline；pyhsmm 负责 duration prior。

### 2. `BBN node/branch posterior update`

先看：

* **statsmodels regime_switching**
* **dtolpin/bocd**
* **hildensia/bayesian_changepoint_detection**

建议实现两个 posterior 通道：

```text
regime_posterior_t = HamiltonFilter(evidence_t)

break_posterior_t = BOCPD(run_length_t, evidence_t)
```

然后：

```text
branch_birth_score =
    transition_prior(node_i -> node_j)
  * break_posterior_t
  * evidence_likelihood_t
```

### 3. `artifact-validation prior source`

先看：

* **cran/powerprior**
* **ethan-alt/hdbayes**

```text
prior_alpha[path] += source_weight * quality_score * offline_success_count
prior_beta[path]  += source_weight * quality_score * offline_failure_count
```

```text
a_source = base_source_weight

```

### 4. `live feedback posterior update`

先看：

* **Toloka/crowd-kit**

你可以把 source reliability 学出来，而不是永远手写：

```text
P(true_outcome | source_votes)
```

其中 source 是：

```text
live_user_feedback
artifact_validation
backtest
research
mutation
```

### 4. `regime_filter.rs`

Hamilton filter 直接自己写，没必要依赖大库。

```rust
pub fn hamilton_filter_step(
    prev_belief: &[f64],
    transition: &[Vec<f64>],
    likelihood: &[f64],
) -> Vec<f64> {
    let n = prev_belief.len();
    let mut pred = vec![0.0; n];

    for j in 0..n {
        for i in 0..n {
            pred[j] += transition[i][j] * prev_belief[i];
        }
        pred[j] *= likelihood[j];
    }

    let z: f64 = pred.iter().sum();
    if z <= 0.0 {
        return vec![1.0 / n as f64; n];
    }

    pred.into_iter().map(|x| x / z).collect()
}
```

```rust
pub fn ips_weight(
    execution_propensity: f64,
    compliance_propensity: f64,
    clip: f64,
) -> f64 {
    let p = execution_propensity * compliance_propensity;
    if p <= 0.0 {
        return 0.0;
    }
    (1.0 / p).min(clip)
}
```

```rust
        return 1.0;

}
```
